⚔️ Nível 1: Desafio

A Jornada do(a) Herói(na) de Algorithmia

🎲 AVENTURA ÉPICA DE PROGRAMAÇÃO

Desperte o Mestre de Códigos que existe em você!

Nesta atividade, você colocará em prática todos os conceitos aprendidos até aqui, criando um personagem em um universo de fantasia medieval com elementos tecnológicos.
Você irá programar a ficha do seu herói ou heroína e gerar automaticamente sua lore — uma narrativa descritiva e épica baseada nos dados que você criou.
  • Declaração de variáveis
  • Declaração de constantes
  • Uso de operadores de atribuição
  • Uso de operadores aritméticos
  • Uso de operadores lógicos
📌 Importante: Não utilize estruturas condicionais (if) nesta atividade.

🧩 Etapas da Aventura

1🔹 Criação das Variáveis Principais
Utilizando let, declare e atribua valores coerentes às seguintes informações do personagem: 
nome        // Nome do personagem
classe      // Tipo de herói (guerreiro, maga, engenheira arcana, etc.)
nivel       // Nível do herói (número entre 1 e 20, por exemplo)
vida        // Pontos de vida
ouro        // Quantidade de moedas
xp          // Experiência acumulada
2🔸 Definição de Constantes Mágicas
Utilize const para representar os equipamentos mágicos do personagem. Esses valores não podem mudar: 
NOME_ARMA       // Ex: "Espada Flamejante"
DANO_BASE       // Ex: 40
NOME_ARMADURA   // Ex: "Capa de Dados"
DEFESA_BASE     // Ex: 20
3⚙️ Aplicação de Operadores de Atribuição
Atualize os valores das variáveis do personagem conforme os eventos abaixo, utilizando operadores de atribuição diferentes:
  • O personagem treinou e ganhou 150 pontos de experiência
  • Comprou uma poção por 30 moedas de ouro
  • Usou a poção e recuperou 40 pontos de vida
  • A arma foi encantada, e seu dano foi dobrado
4➕ Cálculo de Atributos Finais
Com base nas informações anteriores, crie duas variáveis com os seguintes cálculos: 
ataqueTotal = nivel + DANO_BASE
defesaTotal = DEFESA_BASE + (nivel / 2)
5🔐 Avaliação de Prontidão com Operadores Lógicos
Crie variáveis booleanas para verificar as seguintes condições: 
vidaSuficiente → vida > 70
ataqueForte → ataqueTotal > 60
nivelAvancado → nivel >= 10
podeEnfrentarGuardiao → vidaSuficiente && (ataqueForte || nivelAvancado)
Essas variáveis devem ser exibidas no console para mostrar true ou false.
6📜 Geração da Lore do Personagem
Por fim, use console.log() para escrever uma narrativa de aproximadamente 10 linhas, no estilo RPG medieval com elementos tecnológicos. Ela deve usar os dados criados dinamicamente para contar a história do personagem. 
console.log("🧾 LORE DO HERÓI: Kael");
console.log("");
console.log("Kael, o Tecno-Mago errante, percorre as ruínas criptografadas do Reino de Algorithmia.");
console.log("Portando seu Catalisador de Nuvem, canaliza tempestades de código contra inimigos do sistema.");
console.log("Com nível 12 e um ataque total de 92, é temido até mesmo pelos bugs ancestrais.");
console.log("Suas defesas, impulsionadas pela Capa de Dados, atingem 26 pontos de resistência.");
console.log("Após intensos treinamentos, acumula 540 de XP e carrega 90 moedas, pronto para negociações nas tavernas de byte.");
console.log("Restaurado com 85 pontos de vida, Kael é considerado uma das maiores promessas da Geração Binária.");
console.log("Vida suficiente? true | Ataque forte? true | Nível avançado? true");
console.log("Pode enfrentar o Guardião? true — a batalha final se aproxima, e o destino está sendo compilado.");
  • O código deve conter todas as etapas (1 a 6).
  • A lore deve ser gerada dinamicamente a partir dos dados criados.
  • Capriche na criatividade e na apresentação no console.
  • Evite copiar a lore do exemplo — crie a sua própria história!
  • A entrega deve ser feita pelo classroom.
  • Você deve enviar o código como arquivo .js
  • E, você deve enviar o código no formato de imagem(print) ou PDF

⚔️ Nível 2: A Batalha dos Algoritmos

Estruturas Condicionais e Versionamento Arcano

⚔️ BATALHAS ÉPICAS DE CÓDIGO

Domine as artes do combate algorítmico!

Após dominar os fundamentos no Nível 1, agora você enfrentará verdadeiras batalhas algorítmicas! Nesta aventura, aprenderá a tomar decisões estratégicas em combate usando estruturas condicionais.
Além disso, você dominará as artes do Git Arcano — o sistema de versionamento que permite aos mestres programadores preservar e compartilhar seus códigos sagrados através dos tempos.
  • 3.4. Estruturas Condicionais
  • 3.4.1. Condicionais Simples (if)
  • 3.4.2. Condicionais Compostas (if/else)
  • 3.4.3. Condicionais Encadeadas (if/else if/else)
  • Git Básico - Versionamento de Códigos
  • Operadores de comparação avançados
  • Lógica de decisão em combate

🧩 Etapas da Aventura

1🗡️ Preparação para o Combate
Primeiro, crie as variáveis básicas do seu guerreiro. Utilize os conhecimentos do Nível 1 para estabelecer os atributos iniciais que serão usados nas batalhas: 
// Atributos do Guerreiro
let nomeGuerreiro = "Seu Nome Aqui";
let vidaAtual = 100;
let manaAtual = 50;
let nivel = 5;
let ouro = 150;
let experiencia = 0;

// Atributos do Inimigo
let nomeInimigo = "Orc Sombrio";
let vidaInimigo = 80;
let poderInimigo = 25;
2⚔️ Condicionais Simples - A Primeira Decisão
3.4.1. Condicionais Simples: Use a estrutura if para tomar uma única decisão. Se a condição for verdadeira, execute uma ação específica.
Crie verificações simples para diferentes situações de combate: 
// Verificação de vida baixa
if (vidaAtual <= 30) {
    console.log("🚨 ATENÇÃO! Vida crítica! Use uma poção!");
}

// Verificação de mana insuficiente  
if (manaAtual < 20) {
    console.log("🔵 Mana baixa! Não é possível usar magia!");
}

// Verificação de vitória sobre inimigo
if (vidaInimigo <= 0) {
    console.log("⚔️ " + nomeInimigo + " foi derrotado!");
    experiencia += 50;
    ouro += 25;
}
3🛡️ Condicionais Compostas - Escolhas Estratégicas
3.4.2. Condicionais Compostas: Use if/else para criar duas possibilidades mutuamente exclusivas. Uma ação para quando a condição é verdadeira, outra para quando é falsa. 
// Sistema de ataque ou defesa
if (vidaAtual > 50) {
    console.log("💥 Atacando com força total!");
    vidaInimigo -= 30;
    console.log(nomeInimigo + " recebeu 30 de dano!");
} else {
    console.log("🛡️ Modo defensivo ativado!");
    console.log("Recuperando 20 pontos de vida...");
    vidaAtual += 20;
}

// Sistema de recompensas
if (experiencia >= 100) {
    console.log("🎉 LEVEL UP! Você subiu de nível!");
    nivel++;
    experiencia = 0;
    vidaAtual = 100; // Vida restaurada
} else {
    console.log("📊 Continue lutando para ganhar mais XP!");
    console.log("XP atual: " + experiencia + "/100");
}
4🎭 Condicionais Encadeadas - Táticas Avançadas
3.4.3. Condicionais Encadeadas: Use if/else if/else para criar múltiplas possibilidades. Cada condição é testada em ordem até uma ser verdadeira. 
// Sistema de combate inteligente baseado na situação
if (vidaAtual <= 20) {
    console.log("🆘 EMERGÊNCIA! Fugindo do combate!");
    console.log("Você escapou, mas perdeu 10 moedas de ouro.");
    ouro -= 10;
} else if (manaAtual >= 30 && vidaInimigo > 50) {
    console.log("✨ MAGIA PODEROSA! Lançando Bola de Fogo!");
    vidaInimigo -= 40;
    manaAtual -= 30;
    console.log(nomeInimigo + " recebeu 40 de dano mágico!");
} else if (vidaAtual > 60) {
    console.log("⚔️ Ataque físico devastador!");
    vidaInimigo -= 25;
    console.log("Golpe certeiro! " + nomeInimigo + " recebeu 25 de dano!");
} else {
    console.log("🛡️ Tática defensiva! Recuperando forças...");
    vidaAtual += 15;
    manaAtual += 10;
    console.log("Vida e mana parcialmente restauradas!");
}
5🎮 Sistema de Batalha Completo
Agora combine tudo em um sistema de batalha épico! Crie uma sequência lógica que use diferentes tipos de condicionais: 
console.log("🏟️ === ARENA DE COMBATE ALGORÍTMICA ===");
console.log("Guerreiro: " + nomeGuerreiro + " VS " + nomeInimigo);
console.log("");

// Condicional Simples - Verificação inicial
if (nivel < 3) {
    console.log("⚠️ Aviso: Você tem nível baixo para esta batalha!");
}

// Condicional Encadeada - Primeira rodada de combate
if (manaAtual >= 40) {
    console.log("🌟 ATAQUE ESPECIAL: Raio Divino!");
    vidaInimigo -= 50;
    manaAtual -= 40;
} else if (vidaAtual > 70) {
    console.log("⚔️ Ataque corpo a corpo!");
    vidaInimigo -= 30;
} else {
    console.log("🛡️ Posição defensiva!");
    vidaAtual += 10;
}

// Condicional Composta - Resultado da batalha
if (vidaInimigo <= 0) {
    console.log("🎉 VITÓRIA ÉPICA!");
    console.log("Você derrotou " + nomeInimigo + "!");
    experiencia += 75;
    ouro += 50;
    console.log("Ganhou: 75 XP e 50 moedas de ouro!");
} else {
    console.log("⚔️ A batalha continua...");
    console.log("Vida do inimigo restante: " + vidaInimigo);
}
6📜 Git Arcano - Versionamento de Códigos Sagrados
Todo mestre programador precisa dominar as artes do Git - o sistema ancestral que preserva e versiona códigos através do tempo e espaço! 
# 🎯 Inicializar um repositório mágico
git init

# 📋 Verificar o status dos seus pergaminhos
git status

# ➕ Adicionar arquivos ao grimório de preparação
git add nome-do-arquivo.js
git add .  # Adiciona todos os arquivos

# 💾 Criar um checkpoint na sua jornada
git commit -m "Implementei sistema de batalha épico"

# 🌐 Conectar com repositório remoto
git remote add origin https://github.com/seu-usuario/projeto.git

# 🚀 Enviar códigos para o reino digital
git push origin main

# 📥 Baixar códigos do reino digital  
git pull origin main

# 🌿 Criar uma nova linha temporal
git branch nova-funcionalidade
git checkout nova-funcionalidade
🎯 Sua missão Git: Crie um repositório no GitHub chamado "batalha-dos-algoritmos" e faça o versionamento do seu código de batalha!
7📖 Narrativa Épica Final
Crie uma narrativa de aproximadamente 15 linhas que conte a história da sua batalha usando as condicionais. A história deve reagir dinamicamente aos valores das suas variáveis! 
console.log("📜 === CRÔNICAS DA BATALHA ALGORÍTMICA ===");
console.log("");

// Narrativa que muda baseada nas condições
if (nivel >= 5) {
    console.log("O experiente " + nomeGuerreiro + ", veterano de muitas batalhas...");
} else {
    console.log("O jovem " + nomeGuerreiro + ", em sua primeira grande aventura...");
}

console.log("Adentrou as Masmorras do Código, onde o temível " + nomeInimigo + " aguardava.");

if (manaAtual >= 40) {
    console.log("Com poder mágico suficiente, invocou os ancient scripts dos céus!");
} else if (vidaAtual > 70) {
    console.log("Confiando em sua força física, empunhou a espada do debugging!");
} else {
    console.log("Cauteloso, adotou uma postura defensiva e estratégica!");
}

if (vidaInimigo <= 0) {
    console.log("A batalha terminou em glória! O " + nomeInimigo + " foi banido para o /dev/null!");
    console.log(nomeGuerreiro + " emergiu vitorioso, com " + experiencia + " pontos de sabedoria!");
} else {
    console.log("A luta épica continua... O destino ainda será decidido!");
}

console.log("E assim, mais um capítulo foi escrito no Git da história...");
  • Implemente pelo menos 3 condicionais simples (if)
  • Crie pelo menos 2 condicionais compostas (if/else)
  • Desenvolva pelo menos 1 condicional encadeada (if/else if/else)
  • Crie um repositório Git e faça pelo menos 3 commits
  • A narrativa deve usar condicionais para reagir aos dados
  • Demonstre criatividade na história e lógica de combate
  • Entregue o link do repositório GitHub + arquivo .js
  • Inclua print/PDF do código funcionando
📌 Dica Especial: Use comentários no código para explicar sua lógica de batalha! Isso mostra domínio dos conceitos e facilita a correção.
