A Jornada do(a) Herói(na) de Algorithmia

🎲 AVENTURA ÉPICA DE PROGRAMAÇÃO

Desperte o Mestre de Códigos que existe em você!

Nesta atividade, você colocará em prática todos os conceitos aprendidos até aqui, criando um personagem em um universo de fantasia medieval com elementos tecnológicos.
Você irá programar a ficha do seu herói ou heroína e gerar automaticamente sua lore — uma narrativa descritiva e épica baseada nos dados que você criou.
  • Declaração de variáveis
  • Declaração de constantes
  • Uso de operadores de atribuição
  • Uso de operadores aritméticos
  • Uso de operadores lógicos
📌 Importante: Não utilize estruturas condicionais (if) nesta atividade.

🧩 Etapas da Aventura

1🔹 Criação das Variáveis Principais
Utilizando let, declare e atribua valores coerentes às seguintes informações do personagem: 
nome        // Nome do personagem
classe      // Tipo de herói (guerreiro, maga, engenheira arcana, etc.)
nivel       // Nível do herói (número entre 1 e 20, por exemplo)
vida        // Pontos de vida
ouro        // Quantidade de moedas
xp          // Experiência acumulada
2🔸 Definição de Constantes Mágicas
Utilize const para representar os equipamentos mágicos do personagem. Esses valores não podem mudar: 
NOME_ARMA       // Ex: "Espada Flamejante"
DANO_BASE       // Ex: 40
NOME_ARMADURA   // Ex: "Capa de Dados"
DEFESA_BASE     // Ex: 20
3⚙️ Aplicação de Operadores de Atribuição
Atualize os valores das variáveis do personagem conforme os eventos abaixo, utilizando operadores de atribuição diferentes:
  • O personagem treinou e ganhou 150 pontos de experiência
  • Comprou uma poção por 30 moedas de ouro
  • Usou a poção e recuperou 40 pontos de vida
  • A arma foi encantada, e seu dano foi dobrado
4➕ Cálculo de Atributos Finais
Com base nas informações anteriores, crie duas variáveis com os seguintes cálculos: 
ataqueTotal = nivel + DANO_BASE
defesaTotal = DEFESA_BASE + (nivel / 2)
5🔐 Avaliação de Prontidão com Operadores Lógicos
Crie variáveis booleanas para verificar as seguintes condições: 
vidaSuficiente → vida > 70
ataqueForte → ataqueTotal > 60
nivelAvancado → nivel >= 10
podeEnfrentarGuardiao → vidaSuficiente && (ataqueForte || nivelAvancado)
Essas variáveis devem ser exibidas no console para mostrar true ou false.
6📜 Geração da Lore do Personagem
Por fim, use console.log() para escrever uma narrativa de aproximadamente 10 linhas, no estilo RPG medieval com elementos tecnológicos. Ela deve usar os dados criados dinamicamente para contar a história do personagem.

3⚔️ Condicionais Simples - A Primeira Decisão
3.4.1. Condicionais Simples: Use a estrutura if para tomar uma única decisão. Se a condição for verdadeira, execute uma ação específica.
Na sua história: Crie verificações únicas que desencadeiam eventos especiais na narrativa: 
// CAPÍTULO 1: Condicionais Simples na narrativa
console.log("🌅 CAPÍTULO 1: O Aviso dos Anciões");

// Verificação baseada no nível do personagem
if (nivel < 3) {
    console.log("⚠️ Um ancião o adverte: 'Você ainda é jovem demais para esta missão!'");
    console.log("Mas " + nomePersonagem + " decide prosseguir mesmo assim...");
}

// Verificação de recursos especiais
if (ouro >= 100) {
    console.log("💰 Suas moedas tilintam no bolso, atraindo olhares cobiçosos...");
}

// Verificação de classe específica
if (classe === "Mago") {
    console.log("✨ A energia mágica flui através de suas veias...");
}
4🛡️ Condicionais Compostas - Escolhas Estratégicas
3.4.2. Condicionais Compostas: Use if/else para criar duas possibilidades mutuamente exclusivas na sua narrativa.
Na sua história: Crie momentos de escolha binária com caminhos distintos: 
// CAPÍTULO 2: Escolha Estratégica
console.log("🛡️ CAPÍTULO 2: A Encruzilhada do Destino");

// Escolha baseada em recursos
if (ouro >= 50) {
    console.log("💰 Com ouro suficiente, " + nomePersonagem + " compra equipamentos melhores!");
    forca += 5;
    defesa += 3;
    ouro -= 50;
    console.log("Força e defesa aumentaram! Ouro restante: " + ouro);
} else {
    console.log("💔 Sem ouro suficiente, " + nomePersonagem + " deve confiar apenas em suas habilidades!");
    console.log("A adversidade fortalece o espírito! Agilidade +2");
    agilidade += 2;
}

// Sistema de recompensas narrativo
if (experiencia >= 100) {
    console.log("🎉 A sabedoria acumulada se manifesta! " + nomePersonagem + " sente-se mais poderoso!");
    nivel++;
    experiencia = 0;
    vidaAtual = vidaMaxima; // Vida restaurada
} else {
    console.log("📊 " + nomePersonagem + " ainda busca mais conhecimento e experiência...");
    console.log("Sabedoria atual: " + experiencia + "/100");
}
5🎭 Condicionais Encadeadas - Táticas Avançadas
3.4.3. Condicionais Encadeadas: Use if/else if/else para criar múltiplas possibilidades complexas na sua narrativa.
Na sua história: Implemente sistemas complexos com múltiplas possibilidades: 
// CAPÍTULO 3: A Batalha Decisiva
console.log("⚔️ CAPÍTULO 3: A Batalha Decisiva");
let poderInimigo = 30;

// Sistema de combate inteligente baseado na situação
if (vidaAtual <= 30) {
    console.log("🆘 Com pouca vida, " + nomePersonagem + " usa uma tática desesperada!");
    console.log("Ataque final com toda a força restante!");
    poderInimigo -= (forca * 2);
} else if (manaAtual >= 30 && classe === "Mago") {
    console.log("✨ " + nomePersonagem + " canaliza todo seu poder mágico!");
    console.log("Magia devastadora!");
    poderInimigo -= (forca + 20);
    manaAtual -= 30;
} else if (agilidade >= 15) {
    console.log("🏃‍♂️ Com grande agilidade, executa um ataque certeiro!");
    poderInimigo -= forca;
} else {
    console.log("🛡️ Luta defensiva e calculada!");
    poderInimigo -= (forca / 2);
    vidaAtual += 10; // Recupera um pouco de vida
}

// === CAPÍTULO 2: O RITUAL DA EXPLORAÇÃO ===

// === CAPÍTULO 3: A UNIÃO DOS ALIADOS ===

} else { // Demais aliados - Especialistas

}
